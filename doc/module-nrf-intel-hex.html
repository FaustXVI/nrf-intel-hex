    (inner) arraysToHex(blocks, lineSizeopt) → {String}

    Takes a Map of Uint8Arrays as a parameter, and returns a String of text representing a .hex file.
    The input data structure is equivalent to the output of hexToArrays.
    The writer has an opinionated behaviour. Check the project's README file for details.

    (inner) hexToArrays(hexText, maxBlockSizeopt)

    Parses a string containing data formatted in "Intel HEX" format, and returns a Map of Uint8Arrays. In every entry of the Map, the key is the starting address of a data block, and the value is the Uint8Array with the data for that block.
    A .hex file can contain a single block of contiguous binary data starting at memory address 0 (and it's the common case for simple .hex files), but complex files with several non-contiguous data blocks are also possible, thus the need for a data structure on top of the Uint8Arrays.
    The insertion order of keys in the Map is guaranteed to be strictly ascending. In other words, when iterating through the Map, the addresses will be ordered in ascending order.
    The parser has an opinionated behaviour, and will throw a descriptive error if it encounters some malformed input. Check the project's README file for details.

    Example
    import { hexToArrays } from 'nrf-intel-hex';

let intelHexString =
    ":100000000102030405060708090A0B0C0D0E0F1068\n" +
    ":00000001FF";

let byteArrays = hexToArrays(intelHexString);

for (let [address, dataBlock] of byteArrays) {
    console.log('Data block at ', address, ', bytes: ', dataBlock);
}
